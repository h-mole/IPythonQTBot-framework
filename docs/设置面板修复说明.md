# 设置面板修复说明

## 修复内容

### 1. ✅ 设置面板改为独立 Dialog 弹窗

**问题**: 原来的 `SettingsPanel` 是基于 `QWidget` 的非模态窗口，用户体验不够专注。

**解决方案**: 
- 将 `SettingsPanel` 重命名为 `SettingsDialog`
- 继承自 `QDialog` 而非 `QWidget`
- 使用模态方式显示 (`exec()`)
- 添加关闭按钮

**改动文件**:
- `app_qt/widgets/settings_panel.py`: 重写为 QDialog
- `app_qt/main_window.py`: 使用 `exec()` 方法显示对话框
- `app_qt/widgets/__init__.py`: 更新导出名称
- `single_component_tests/test_settings_panel.py`: 更新测试代码
- `docs/设置面板使用指南.md`: 更新文档

### 2. ✅ 简化数据结构，移除重复的 MainAppConfig

**问题**: `MainAppConfig` 和 `MainAppConfigSettings` 功能重复，造成代码冗余。

**解决方案**:
- 保留 `MainAppConfigSettings` 作为主要配置类（支持动态表单）
- 简化 `MainAppConfig` 仅用于兼容旧代码
- 默认配置文件生成改为直接生成 JSON，不再通过 Pydantic 模型

**改动文件**:
- `app_qt/configs.py`: 
  - 删除 `MainAppConfigSettings.create_default_config_json_str()` 方法
  - 修改 `ensure_app_config_file` 使用字典生成默认配置
  - 保留 `app_config` 全局变量以避免破坏现有依赖

## 技术细节

### SettingsDialog 关键特性

```python
class SettingsDialog(QDialog):
    def __init__(self, settings: MainAppConfigSettings, parent=None):
        super().__init__(parent)
        
        # 设置为模态对话框
        self.setWindowModality(Qt.ApplicationModal)
        
        # 设置标题和尺寸
        self.setWindowTitle("⚙️ 系统设置")
        self.setMinimumSize(800, 600)
        self.resize(900, 700)
```

### 使用方式

```python
# 从菜单打开
def open_settings_panel(self):
    from app_qt.configs import settings as global_settings
    
    # 创建并显示对话框（模态）
    self.settings_dialog = SettingsDialog(global_settings, self)
    self.settings_dialog.exec()  # 阻塞直到用户关闭
```

### 数据结构对比

**之前（重复）**:
```python
# 两个几乎相同的类
class LLMConfig(BaseModel):
    provider: str
    model: str
    ...

class LLMConfigSettings(BaseSettings):
    provider: str = Field(...)
    model: str = Field(...)
    ...
```

**现在（统一）**:
```python
# 只保留一个主要类
class LLMConfigSettings(BaseSettings):
    provider: str = Field(default="", title="Provider")
    model: str = Field(default="", title="Model")
    ...

# MainAppConfig 仅用于兼容
class MainAppConfig(BaseModel):
    llm_config: LLMConfig  # 简单包装
```

## 注意事项

1. **模态 vs 非模态**: 
   - 设置对话框是模态的（阻塞主窗口）
   - 未配置提示是非模态的（不阻塞）

2. **配置文件位置**: 
   - `config.json` 在项目根目录
   - 首次运行会自动创建

3. **兼容性**:
   - `app_config` 仍然可用
   - 新代码建议使用 `settings` 实例
