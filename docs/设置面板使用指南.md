# 设置面板使用指南

## 功能概述

设置面板是一个独立的配置管理界面，允许用户通过图形界面轻松管理系统配置，特别是 LLM 提供商配置。

## 主要特性

### 1. 独立的设置对话框
- **位置**: `app_qt/widgets/settings_panel.py`
- **组件**: `SettingsDialog`
- **特点**:
  - 动态表单生成（基于 `pyside6-settings`）
  - 实时配置编辑
  - 保存/重置功能
  - 模态窗口设计（阻塞主窗口，专注于配置）
  - 独立弹窗形式

### 2. 未配置提示对话框
- **组件**: `UnconfiguredDialog`
- **特点**:
  - 非模态提示窗口
  - 不阻塞程序加载
  - 自动引导用户到设置面板
  - 友好的配置说明

### 3. 菜单栏集成
- **路径**: 工具 → 设置 (🔧 → ⚙️ 设置)
- **快捷键**: `Ctrl+,`
- **功能**: 快速打开设置面板

## 使用方法

### 从菜单打开设置面板

1. 启动主程序
2. 点击菜单栏的 **🔧 工具**
3. 选择 **⚙️ 设置**
4. 设置对话框将弹出（模态窗口）

### 快捷键方式

按下 `Ctrl+,` 直接打开设置对话框

### 自动提示配置

如果程序检测到您还没有配置 LLM 提供商：
1. 会自动显示一个非模态提示对话框
2. 对话框会说明需要配置的内容
3. 点击 **"立即配置"** 按钮会自动打开设置对话框
4. 您也可以点击 **"稍后配置"** 暂时关闭提示

## 配置步骤

### 首次配置 LLM 提供商

1. 打开设置面板
2. 找到 **LLM Config** 部分
3. 展开 **Provider List**（提供商列表）
4. 点击 **+** 按钮添加新提供商
5. 填写以下信息：
   - **Name**: 提供商名称（如 "OpenAI"、"Azure"）
   - **API Key**: 您的 API 密钥
   - **API URL**: API 端点 URL
6. 返回上级，在 **Provider** 下拉框中选择刚才添加的提供商
7. 在 **Model** 中输入模型名称
8. 点击顶部的 **💾 保存配置** 按钮

### 修改现有配置

1. 打开设置对话框（`Ctrl+,`）
2. 导航到要修改的配置项
3. 直接编辑相应字段
4. 点击 **💾 保存配置**

### 重置配置

1. 打开设置对话框
2. 点击 **🔄 重置** 按钮
3. 确认重置操作
4. 配置将恢复到上次保存的状态

### 配置检查逻辑

在 `main_window.py` 中：

```python
def _check_initial_configuration(self):
    """检查初始配置状态，如果未配置则显示提示"""
    from app_qt.configs import settings as global_settings
    
    if not check_and_show_unconfigured_dialog(global_settings, self):
        # 如果未配置，延迟 500ms 自动打开设置对话框
        QTimer.singleShot(500, self.open_settings_panel)
```

### 打开设置对话框

在 `main_window.py` 中：

```python
def open_settings_panel(self):
    """打开设置对话框"""
    from app_qt.configs import settings as global_settings
    
    # 如果对话框已经存在，直接显示
    if self.settings_dialog and self.settings_dialog.isVisible():
        self.settings_dialog.raise_()
        self.settings_dialog.activateWindow()
        return
    
    # 创建新的设置对话框（模态）
    self.settings_dialog = SettingsDialog(global_settings, self)
    self.settings_dialog.exec()  # 模态执行
```

### 全局 Settings 实例

在 `configs.py` 中创建了全局 settings 实例：

```python
# 创建全局设置实例（用于设置面板）
settings = MainAppConfigSettings.load(config_file="config.json", auto_create=True)
```

这样所有模块都可以使用同一个 settings 实例：

```python
from app_qt.configs import settings

# 访问配置
provider = settings.llm_config.provider

# 保存配置
settings._save_settings()
```

### 数据结构设计

为了简化代码结构，只保留了一个主要的配置类：

- **MainAppConfigSettings**: 用于设置界面的动态表单生成
- **MainAppConfig**: 保留以兼容旧代码（不推荐使用）

这样的好处是：
1. 减少重复代码
2. 统一数据源
3. 避免同步问题

## 常见问题

### Q: 设置面板打不开怎么办？
A: 检查控制台输出，查看是否有错误信息。如果配置文件损坏，可能需要手动编辑 `config.json`。

### Q: 配置保存在哪里？
A: 配置保存在项目根目录的 `config.json` 文件中。

### Q: 如何备份配置？
A: 直接复制 `config.json` 文件即可。

### Q: 可以在代码中访问设置吗？
A: 可以，通过 `from app_qt.configs import settings` 导入全局 settings 实例。

## 扩展开发

### 添加新的配置项

1. 在 `configs.py` 中的对应类里添加新字段
2. 使用 `Field()` 定义字段的元数据（标题、默认值等）
3. 重新运行程序，新字段会自动出现在设置面板中

示例：
```python
class LLMConfigSettings(BaseSettings):
    provider: str = Field(default="", title="Provider")
    new_setting: str = Field(default="default", title="New Setting")  # 新配置项
```

### 自定义设置面板

如果需要自定义 UI，可以继承 `SettingsPanel`：

```python
class CustomSettingsPanel(SettingsPanel):
    def _init_ui(self):
        super()._init_ui()
        # 添加自定义 UI 元素
```

## 注意事项

1. **配置文件格式**: 确保 `config.json` 是有效的 JSON 格式
2. **API Key 安全**: 不要将包含真实 API Key 的配置文件提交到版本控制系统
3. **保存操作**: 修改配置后记得点击“保存配置”按钮
4. **模态对话框**: 设置对话框是模态的，关闭前无法操作主窗口
5. **非模态提示**: 未配置提示是非模态的，不会阻止程序其他功能的使用
